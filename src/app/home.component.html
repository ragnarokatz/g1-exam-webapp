Routing scenarios

Template version 1

Features

This version 1 of the template includes these features:

  • Bootstrap 3
  • The routing feature
  • A navigation menu
  • Some getting-started components

This app uses an in-memory array of data for simplicity.

It implements "get all", "get one", and "add new".

A data model manager service is used to manage the data and its data service operations. Each component will call a service method to do its task.

By design, "edit existing" and "delete item" are deferred until later, because they work quite differently when we have a database involved, so there's no point in covering them now.

Description

This app shows the typical routing scenarios:

  • Basic essential routing configuration
  • Routing with parameters
  • Extracting and using route (URL) data

The basic essential routing config is in app-routing.module.ts.

In this source code file, it also shows the technique we use to route with parameters, to support URL segments like /vehicles/345.

In a component that needs to use route (URL) data, we can extract and use the data we need, typically an identifier. We can also use this data to programmatically navigate.
